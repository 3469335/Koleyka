## Технические детали реализации

### API Routes для создания
1. `/api/auth/check-user` - проверка пользователя по номеру
2. `/api/zapis` - CRUD операции для zapis (с проверкой прав)
3. `/api/zapis/count` - подсчет записей в очереди
4. `/api/zapis/my` - получение записи для User4

### Страницы для создания
1. `/login` или `/` - страница входа
2. `/zapis` - страница таблицы zapis (для User2, User3)
3. `/zapis/my` - страница для User4
4. `/view-db/tables` - страница списка таблиц (для User1)

### Компоненты для создания
1. `LoginForm` - форма входа
2. `ZapisTable` - таблица zapis с подсветкой
3. `ZapisForm` - форма создания/редактирования zapis
4. `QueueCounter` - компонент подсчета записей в очереди
5. `UserInfo` - отображение информации о текущем пользователе

### Утилиты для создания
1. `lib/auth.ts` - функции для работы с аутентификацией
2. `lib/permissions.ts` - проверка прав доступа
3. `lib/queue-calculator.ts` - подсчет записей в очереди
4. `lib/row-styles.ts` - определение стилей строк по статусу
