Проект Koleyka 

1. основная информация находится в таблице zapis 
2. при выводе  таблицы zapis вверху страницы показывается количество авто в очереди (общее количество записей за вычетом записей со status "недозвон 2", "отказ" и "заехал").
3. строки со status "заехал" подсвечиваются зелёным
4. строки со status "едет" подсвечиваются светло-зелёным
5. строки со status "недозвон 2" и "отказ" подсвечиваются синим
6. строки со status "ремонт" и  подсвечиваются желтым


работает по следуюшей логике:

1. При старте проекта пользователь видит поле с предложением "Введите номер"
2. На основании номера определяется тип пользователя по таблице Users, если номера в таблице Users нет, то выдать сообщение, что "Номера нет в БД"
3. Для User1 доступны все действия CRUD со всеми таблицами, он направляется на страницу со списком таблиц БД
4. Для User2 открывается таблица zapis с правом просмотра. 
5. Для User3 открывается таблица zapis с правами просмотра, редактирования и добавления записей. При добавлении новой строки в таблицу zapis при заполнении поля name создается новый пользователь типа User4 с именем zapisName.
6. Для User4 открывается строка zapis с его name с правом просмотра. Вверху строки показывается количество записей перед ним без учёта имеющих status "недозвон 2", "отказ" и "заехал".
